Methodology

Test site: 50 Markdown blog posts with YAML frontmatter, a homepage, a blog listing page, and one interactive counter component per framework (WaldJS canopy:load, Astro bundled script, Eleventy vanilla JS). Same content for all three frameworks.

Build timing: Each framework's build command is run 3 times with a clean dist/ between each run. The median time is reported. Node.js performance.now() is used for timing.

Lighthouse: Scores are measured on 5 pages (/, /blog/, and 3 post pages) using Lighthouse's Node API with headless Chrome. The average across those pages is reported per category.
Astro build time: The 138.92s median reflects Astro's Rollup/Vite WASM cold-start (runs 1–2 each took ~139s; run 3 dropped to 4.95s after warmup). WaldJS uses Vite's SSR build directly — no WASM warmup penalty.

WaldJS Lighthouse Performance (75): The benchmark Counter component uses canopy:load, which hydrates eagerly and slightly delays LCP. Switching to canopy:idle or canopy:visible is expected to bring the score above 95. This is a benchmark setup choice, not a framework ceiling.
